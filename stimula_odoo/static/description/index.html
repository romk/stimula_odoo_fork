Stimula for Odoo

Easy data import, export and migration with STML.

STML, or Simple Table Mapping Language, is a standardized language designed for mapping data between CSV files and SQL databases. It forms the foundation for data import and export, enabling data analysts and developers to define fields, transformations, foreign key resolutions, and row selections in a simple, repeatable, and reusable way.

Local Installation

Follow the steps below to install the Stimula for Odoo module on a local Odoo instance:

1. Copy the stimula_odoo folder into your addons folder in Odoo.

2. Restart Odoo.

3. Login to Odoo and activate the stimula_odoo module.

4. Use the following endpoint to verify the module was installed correctly: http://localhost:8069/stimula/1.0/hello

Installation on odoo.sh

Follow the steps below to install the Stimula for Odoo module on Odoo.sh:

1. Add the Stimula for Odoo module to your list of repositories. You can do this by forking or cloning the following repository: https://github.com/stmlio/stimula_odoo

2. Create a project on odoo.sh and trigger a new build. This will automatically activate the module.

3. Login to Odoo and activate the module.

4. Use the following endpoint to verify the module was installed correctly: https://your-database-name.dev.odoo.com

Optional configuration

The module supports the following system configuration parameters:

stimula_odoo.secret_key: This is the secret key to encrypt and sign access tokens. Thee module automatically generates a random secret key during first use.

stimula_odoo.token_lifetime: This is the access token lifetime in seconds. The default value is 24 hours.

Multi tenant setup

This module supports a multi-tenant configuration with multiple databases per Odoo instance.

If you're planning to run multiple databases in your Odoo instance, then make sure that the module’s endpoints are available across all database by adding the module to your list of server-wide modules in odoo.conf. For example as follows:

server_wide_modules = web,stimula_odoo

In a multi-tenant setup, each database has its own secret key. By default the secret keys are different on eacht database, so a user can use a Stimula access token only on a single database. If you’ve added stimula_odoo to your list of server-wide modules, then it is no longer needed to separately activate the module in your database.
